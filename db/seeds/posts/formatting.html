Codidact sites support Markdown for formatting. There are a number of dialects of Markdown, but we use CommonMark as it's the closest thing there is to a standard.

What formatting is available?

  • **bold text** gets you bold text
  • *italics* or _italics_ gets you italics - note that underscore-based italics are not permitted in the middle of words
  • `code` gets you code - used for code, program output, or anything you want monospaced
  • [name](url) gets you name - i.e. name is a link to url
  • ~~strike~~ gets you strike
  • For blockquotes, use a > at the start of your line
  • For bulleted lists, use a * or - at the start of your lines
  • For numbered lists, use 1. and 2. etc. at the start of your lines
  • To get headings, use # at the start (or ## for a second-level heading, ### for third, etc)
  • For a horizontal line, put --- on a line on its own, with another blank line either side
  • For footnotes, use [^1], [^2], etc, inline (i.e. where you want the [1] to appear), then add [^1]: footnote text to the end of your post

What HTML is supported?

You can almost always do all the formatting you need with simple Markdown. However, if you know what you're doing or you want to specify more precisely what you want, you can use HTML. The following tags are supported:

  • a
  • p
  • b, strong
  • i, em
  • hr
  • h1 through h6
  • blockquote
  • img
  • strike, del
  • code, pre
  • br
  • ul, ol, li
  • sup, sub
  • section

All other HTML is sanitized and not displayed.
